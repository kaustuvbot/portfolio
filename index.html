Project 1 — S3 Static Website // Terraform · AWS

Foundational Tier · Completed

Provisioned a fully automated static website hosting solution on AWS S3 using Terraform. Configured bucket policies, static website hosting settings, and public access controls entirely via infrastructure as code.

What Was Built

  • S3 bucket creation with static website hosting enabled via Terraform
  • Bucket policy configuration for public read access
  • Output of the website endpoint URL as a Terraform output value
  • Separated configuration into main.tf, variables.tf, and terraform.tfvars

Key Learnings

  • AWS S3 bucket policies and ACL configuration
  • Terraform file organisation best practices for environment reusability
  • Using terraform output to surface infrastructure endpoints

Project 2 — EC2 Instance with VPC Networking // Terraform · AWS

Foundational Tier · Completed

Provisioned a full VPC networking stack and deployed an EC2 instance using Terraform. Built the entire network layer from scratch — including internet connectivity, routing, and security — without relying on AWS defaults.

Infrastructure Provisioned

  • Custom VPC with defined CIDR block
  • Internet Gateway attached to the VPC for outbound internet access
  • Public Subnet with auto-assign public IP enabled
  • Route Table with a default route to the Internet Gateway, associated to the subnet
  • Security Group allowing SSH (port 22) and HTTP (port 80) inbound traffic
  • EC2 instance on Amazon Linux 2023, deployed into the public subnet

Key Learnings

  • How VPC components interconnect: subnets, route tables, gateways, and security groups
  • Terraform resource dependencies and implicit vs explicit depends_on
  • Using data sources (aws_ami) to dynamically fetch the latest Amazon Linux AMI
  • Terraform variable types, locals, and output blocks
TerraformAWS VPCEC2 Internet GatewaySecurity GroupsRoute Tables
