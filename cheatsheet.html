Linux + Git

Shell work, file inspection, and source control

This section combines everyday terminal commands with the Git commands most often used during local development, review, recovery, and collaboration.

Focus Navigation, search, process inspection, branching, commits, history, and remote sync.

Navigation and File Operations

Move around quickly and create, copy, or remove files safely.

pwd

Print the current working directory.

ls -lah

List files with permissions, sizes, and hidden entries.

cd /path/to/dir

Change into a target directory.

mkdir -p app/config

Create nested directories in one command.

cp -r src/ backup/

Copy a directory recursively.

mv old.txt new.txt

Move or rename a file.

rm -i file.txt

Remove a file with confirmation.

Safer default: prefer `-i` unless you are fully sure.

Inspection and Search

Inspect content, compare files, and search code or logs efficiently.

cat file.txt

Print a small file to stdout.

less /var/log/syslog

Open a long file with paging and search.

tail -f app.log

Follow a log file in real time.

find . -name "*.yaml"

Find matching files under the current directory.

grep -R "DATABASE_URL" .

Recursively search file contents for a string.

diff -u old.conf new.conf

Show a unified diff between two files.

du -sh *

Summarize directory sizes in the current path.

Permissions and Processes

Check ownership, adjust file modes, and inspect running processes.

chmod 644 file.txt

Set owner read/write and group/world read permissions.

chmod +x script.sh

Make a script executable.

chown user:group file.txt

Change file owner and group.

ps aux | grep nginx

Search running processes for a service.

top

Monitor CPU, memory, and active processes interactively.

kill -15 1234

Ask a process to terminate gracefully.

kill -9 1234

Force-kill a stuck process.

Use with care: `-9` skips graceful shutdown hooks.

Networking and System Info

Quick checks for connectivity, open ports, and machine state.

uname -a

Show kernel and system details.

df -h

Display disk usage in human-readable units.

free -h

Show memory and swap usage.

ip addr

List network interfaces and IP addresses.

ss -tulpn

Show listening sockets and owning processes.

curl -I https://example.com

Fetch response headers for a URL.

ping -c 4 8.8.8.8

Send a fixed number of ICMP echo requests.

Git Setup and Daily Status

Check repository state and establish your local identity.

git config --global user.name "Your Name"

Set the default author name for commits.

git config --global user.email "you@example.com"

Set the default author email.

git clone https://github.com/org/repo.git

Clone a remote repository locally.

git status

Show branch, staged, unstaged, and untracked changes.

git status --short

Use the condensed status format for quick scanning.

git remote -v

List configured remotes and their URLs.

Branching and Switching

Move between features, create branches, and keep them current.

git branch

List local branches.

git branch -a

List local and remote-tracking branches.

git switch main

Switch to an existing branch.

git switch -c feature/cheatsheet

Create and switch to a new branch.

git fetch origin

Update remote-tracking references without merging.

git pull --rebase origin main

Replay local commits on top of updated remote history.

git branch -d feature/old-branch

Delete a merged local branch.

Staging and Commits

Stage intent clearly and keep local history readable.

git add file.txt

Stage one file for the next commit.

git add .

Stage tracked and untracked changes under the current directory.

git add -p

Interactively stage only the hunks you want.

git restore --staged file.txt

Unstage a file while keeping its working changes.

git commit -m "Add cheatsheet page"

Create a commit with a message.

git commit --amend

Update the most recent commit.

Caution: avoid amending commits that are already shared.

History, Diff, and Recovery

Inspect changes over time and recover from common mistakes.

git log --oneline --graph --decorate -20

Show a compact graphical view of recent history.

git diff

Compare working tree changes against the index.

git diff --staged

Compare staged changes against the last commit.

git show HEAD

Display the latest commit and patch.

git restore file.txt

Discard unstaged changes in a file.

Destructive: this replaces the working copy of that file.

git reflog

Find recent branch and HEAD movements for recovery.

git reset --soft HEAD~1

Undo the last commit but keep changes staged.

Docker

Images, containers, logs, and cleanup workflows

Use these commands for common image builds, runtime inspection, container debugging, and local environment cleanup.

Focus Build, run, inspect, debug, manage volumes, and work with Compose.

Docker Basics

Start with version checks, image builds, and one-off runs.

docker version

Show Docker client and server versions.

docker info

Display engine configuration and runtime details.

docker build -t myapp:latest .

Build an image from the current directory.

docker run --rm alpine:latest echo "hello"

Run a container once and remove it after exit.

docker login

Authenticate to a container registry.

Images

Inspect and publish images with tags that match deployment intent.

docker images

List images stored locally.

docker image inspect myapp:latest

Show image metadata and configuration JSON.

docker tag myapp:latest registry.example.com/myapp:v1

Create a new tag for an existing image.

docker push registry.example.com/myapp:v1

Push a tagged image to a registry.

docker pull nginx:stable

Download an image from a registry.

Containers

Run, stop, and remove containers without losing track of state.

docker ps

List running containers.

docker ps -a

List all containers, including stopped ones.

docker run -d --name web -p 8080:80 nginx

Run a container in detached mode with a port mapping.

docker stop web

Stop a running container gracefully.

docker start web

Start an existing stopped container.

docker rm web

Remove a stopped container.

Logs and Inspection

See why a container failed and inspect how it was configured.

docker logs web

Print container logs.

docker logs -f web

Follow logs in real time.

docker inspect web

Show full container configuration and state.

docker port web

List port mappings for a container.

docker stats

Stream CPU, memory, and network usage for running containers.

Exec and Debugging

Enter a running container and validate its filesystem or network behavior.

docker exec -it web sh

Open an interactive shell in a running container.

docker exec -it web env

Print container environment variables.

docker cp web:/usr/share/nginx/html ./html-backup

Copy files from a container to the host.

docker top web

Show processes running inside a container.

docker diff web

Show filesystem changes made since the container started.

Volumes and Cleanup

Manage persisted data and remove unused local resources deliberately.

docker volume ls

List Docker-managed volumes.

docker volume inspect myvolume

Show mountpoint and metadata for a volume.

docker system df

Summarize disk usage by images, containers, and volumes.

docker image prune

Remove dangling images.

docker container prune

Remove all stopped containers.

docker system prune -a

Remove unused images, containers, networks, and cache.

Destructive: review what counts as unused before running with `-a`.

Compose Essentials

Operate multi-container local environments with a single manifest.

docker compose up -d

Start services in detached mode.

docker compose down

Stop and remove services, networks, and default resources.

docker compose ps

List services and their container state.

docker compose logs -f api

Follow logs for a named service.

docker compose exec api sh

Open a shell in a running Compose service.

Kubernetes

Cluster context, workloads, and troubleshooting flow

These commands cover the most common `kubectl` workflows for switching contexts, inspecting workloads, debugging pods, managing rollouts, and handling configuration resources.

Focus Context, namespaces, pods, deployments, services, logs, configuration, and rollout control.

Cluster and Context

Validate which cluster and namespace you are targeting before changing anything.

kubectl config get-contexts

List available kubeconfig contexts.

kubectl config current-context

Show the active context.

kubectl config use-context my-cluster

Switch to a different context.

kubectl cluster-info

Display the control plane and core service endpoints.

kubectl get nodes -o wide

List nodes with extended details.

Namespaces

Separate environments and scope commands to the correct namespace.

kubectl get ns

List namespaces.

Quick live changekubectl create namespace staging

Create a namespace.

Persistence: add the namespace manifest to source control if it should exist after cluster rebuilds or environment recreation.

kubectl config set-context --current --namespace=staging

Set the default namespace for the current context.

kubectl get all -n staging

List common workload resources in one namespace.

Pods

Inspect pod state, scheduling, and image usage quickly.

kubectl get pods

List pods in the current namespace.

kubectl get pods -A

List pods across all namespaces.

kubectl get pods -o wide

Show nodes, pod IPs, and extra scheduling details.

kubectl describe pod my-pod

Inspect events, conditions, volumes, and container state.

Quick live changekubectl delete pod my-pod

Delete a pod so its controller can recreate it.

Reboot note: this only restarts the live workload. It does not change the manifest, so any underlying config problem will return on the next restart.

Deployments and Rollouts

Update workloads and monitor rollout health without losing history.

kubectl get deployments

List deployments in the current namespace.

kubectl describe deployment web

Show deployment strategy, replica sets, and events.

Quick live changekubectl scale deployment web --replicas=3

Scale a deployment to a target replica count.

Persistence: update the deployment manifest or Helm/Kustomize values too, otherwise the next apply can overwrite the replica count.

Quick live changekubectl set image deployment/web web=repo/web:v2

Update a container image in a deployment.

Persistence: commit the new image tag to the manifest or GitOps source; otherwise a redeploy or sync can revert it.

kubectl rollout status deployment/web

Watch rollout progress.

kubectl rollout undo deployment/web

Roll back to the previous deployment revision.

Services and Ingress

Trace how traffic is exposed and routed through the cluster.

kubectl get svc

List services in the current namespace.

kubectl describe svc web

Show selectors, endpoints, ports, and type details.

kubectl get ingress

List ingress resources.

kubectl describe ingress web

Inspect ingress rules, hosts, and backend mappings.

Quick live changekubectl port-forward svc/web 8080:80

Forward a local port to a service for testing.

Session only: this is not persistent and stops when the process exits or your connection drops.

Logs and Exec

Investigate application behavior from inside the cluster.

kubectl logs my-pod

Print logs from the first container in a pod.

kubectl logs -f my-pod

Follow pod logs in real time.

kubectl logs my-pod -c app

Read logs from a specific container in a multi-container pod.

kubectl exec -it my-pod -- sh

Open an interactive shell inside a container.

kubectl top pod

Show pod CPU and memory usage if metrics-server is installed.

ConfigMaps and Secrets

Handle application configuration separately from image content.

kubectl get configmap

List ConfigMaps in the current namespace.

kubectl describe configmap app-config

Inspect stored key/value data and metadata.

Quick live changekubectl create configmap app-config --from-file=./config/

Create a ConfigMap from local files.

Persistence: store the ConfigMap manifest or generator input in source control; recreating the cluster will otherwise lose it.

kubectl get secret

List secrets in the current namespace.

Quick live changekubectl create secret generic db-creds --from-literal=username=admin --from-literal=password=change-me

Create a generic secret from literal values.

Persistence: keep the secret definition in your secret-management workflow, otherwise cluster rebuilds or GitOps syncs will not recreate it.

kubectl get secret db-creds -o yaml

Inspect the secret manifest, including base64-encoded data.

Reminder: base64 is encoding, not encryption.

Apply and Troubleshooting

Validate manifests, detect drift, and inspect cluster events during failures.

kubectl apply -f deployment.yaml

Create or update resources from a manifest.

kubectl apply -k overlays/prod

Apply resources managed with Kustomize.

kubectl diff -f deployment.yaml

Preview changes before applying a manifest.

kubectl get events --sort-by=.metadata.creationTimestamp

Review recent cluster events in chronological order.

kubectl api-resources

List available resource types exposed by the API server.

kubectl explain deployment.spec.template.spec.containers

Read built-in schema help for a manifest field path.
